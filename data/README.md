# Data

This directory contains all data-related resources used in the project.

The repository does **not** include raw or processed datasets due to size and licensing constraints.

## Structure

```
data/
├── raw/         # Raw datasets (not versioned)
├── processed/   # Generated datasets (not versioned)
└── sample/      # Small versioned examples for testing
```

---

## `raw/`

This folder is intended to store the original datasets.

For this project, it should contain the Yelp Open Dataset files:

- `yelp_academic_dataset_business.json`
- `yelp_academic_dataset_review.json`
- `yelp_academic_dataset_user.json`

### How to populate

1. Download the Yelp dataset from the official source.
2. Extract the files.
3. Place them in `data/raw/`.


---

## `processed/`

This folder contains datasets generated by preprocessing and pipeline scripts.

Typical files include:

### Dataset
- `yelp_subset.jsonl` — processed dataset used for generation

### Generation outputs
- `generated_arguments_*.jsonl`
- `*_valid.jsonl`
- `*_invalid.jsonl`

### Scoring outputs
- `scored_arguments_*.jsonl`
- `*_summary.json`

### Matrix Factorization (MF)
- `mf_dataset.csv` — training data for MF
- `mf_predictions.json` — predicted scores for user–item pairs

These files are produced by running scripts from the `scripts/` directory.

Example:

```bash
python -m scripts.build_yelp_subset
```

This folder can be regenerated at any time with the Yelp dataset.

---

## `sample/`

This folder contains small **versioned** example datasets.

It is meant to document the expected input format, support quick testing without downloading Yelp, and help with debugging and reproducibility.

Example: `sample_input.jsonl`

These files are intentionally small and are synthetic or manually constructed examples. No original Yelp data is redistributed in this repository.
